# docx2pdf

Бібліотека на чистому Go для конвертації документів Word (.docx) у PDF.

Без cgo, без зовнішнього середовища виконання (ні LibreOffice, ні headless-браузера) —
лише Go.

## docx → PDF

```go
package main

import (
	"log"

	"github.com/h0rn3t/docx2pdf/document"
)

func main() {
	doc, err := document.Open("in.docx")
	if err != nil {
		log.Fatal(err)
	}
	defer doc.Close()

	if err := doc.WritePDF("out.pdf"); err != nil {
		log.Fatal(err)
	}
}
```

Готовий до запуску приклад — у [`examples/docx2pdf`](examples/docx2pdf):

```sh
go run ./examples/docx2pdf in.docx out.pdf
```

## Обмеження точності (MVP)

Поточна конвертація відтворює **читабельний, «пливкий» текст тіла документа — а не
піксель-у-піксель копію Word**. Підтримується вже зараз:

- абзаци й текстові runs, у порядку документа поряд із таблицями;
- явні розриви рядка всередині абзацу (`<w:br/>`, розрив Shift+Enter) починають
  новий рядок у межах абзацу, окремо від розриву сторінки (`w:type="page"`);
- на рівні run — **жирний**, *курсив*, підкреслення, гарнітура, розмір і колір
  шрифту;
- вирівнювання абзацу (ліворуч, по центру, праворуч, за шириною), відступи (лівий,
  правий, першого рядка/висячий через `w:ind`), інтервал до/після, міжрядковий
  інтервал і явні розриви сторінок. Властивості абзацу (`w:jc`, `w:ind`,
  `w:spacing`) і run (жирний/курсив/підкреслення, `rFonts`, розмір, колір)
  резолвляться по кожному полю під час `Open`: спершу inline `w:pPr`/`w:rPr`,
  далі символьний стиль `w:rStyle` (для run), потім стиль абзацу `w:pStyle` (і його
  `w:rPr`) за ланцюгом `w:basedOn` у `styles.xml`, далі `w:docDefaults`, інакше
  типові значення — тож абзаци й runs, оформлені іменованими стилями Word без
  inline-властивостей, теж отримують своє форматування. Вертикальний проміжок
  навколо таблиці створюється інтервалом сусідніх абзаців (в OOXML сама таблиця
  не має властивості інтервалу), тож окремої обробки не потребує;
- кольори тексту, зокрема кольори теми з `word/theme/theme1.xml`: `w:color` з явним
  hex або посиланням `w:themeColor` на слот схеми (`dk1`/`lt1`/`dk2`/`lt2`/
  `accent1`…`accent6`/`hlink`/`folHlink`) резолвиться у колір тексту, а `w:themeTint`/
  `w:themeShade` освітлюють/затемнюють його по яскравості HSL — так само, як Word
  рахує варіанти «Lighter 40%»/«Darker 25%» (напр. accent1 `FF0000` із
  `w:themeTint="99"` → `#FF6666`); `auto`/відсутня тема/невідомий слот → типовий
  чорний;
- шрифти теми: `w:rFonts` з `w:asciiTheme`/`w:hAnsiTheme` (`minorHAnsi`,
  `majorHAnsi`, …) резолвиться через `a:fontScheme` теми — мінорний шрифт для
  тексту, мажорний для заголовків, — тож абзаци без явної гарнітури (типовий
  випадок, коли шрифт заданий лише в `w:docDefaults`) беруть шрифт теми, а не
  типовий; явний `w:ascii`/`w:hAnsi` має пріоритет;
- заливка й виділення: `w:shd` на run, абзаці й комірці — фон `w:fill`, візерунки
  (`clear`, `solid`, `pctN` — відсоткове змішування переднього кольору з фоном),
  посилання на тему (`w:themeFill` з tint/shade), `nil`/`none` → без заливки;
  `w:highlight` (усі 16 назв палітри Word) малюється поверх `w:shd`. Заливка run
  малюється під його текстом (разом із пробілами всередині), заливка абзацу — по
  всій його текстовій області, зокрема на кожній сторінці/колонці, якщо абзац
  розірвано;
- нумеровані й марковані списки з `word/numbering.xml`: `w:numPr` (inline або зі
  стилю абзацу) дає маркер (`decimal`, `lowerLetter`/`upperLetter`,
  `lowerRoman`/`upperRoman`, bullet; `lvlText` з `%1`…`%9`, `lvlOverride` для
  start) із лічильниками у порядку документа (глибші рівні скидаються) плюс
  відступ рівня (типово висячий), тож списки, оформлені у Word, дістають видимі
  маркери, а не лише текст. Юридична нумерація (`w:isLgl`) відображає всі рівні
  маркера десятковими; суфікс (`w:suff`) підтримується у всіх трьох режимах —
  `tab` (типовий; маркер займає ширину висячого відступу, тож текст пункту
  починається рівно на позиції табуляції, як у Word), `space` і `nothing`;
  картинкові маркери (`w:numPicBullet` + `w:lvlPicBulletId`) малюються як
  зображення відповідного розміру. Списки в колонтитулах мають власні лічильники
  й не збивають нумерацію тіла документа;
- зображення: вбудовані рисунки (`w:drawing` — `wp:inline` і `wp:anchor`) та
  застарілі VML-картинки (`w:pict`/`v:imagedata`) вбудовуються у PDF (PNG, JPEG,
  GIF) із розміром із `wp:extent` (EMU) або, за його відсутності, з власних
  пікселів при 96 dpi; картинка сидить на базовій лінії як гліф і задає висоту
  свого рядка. Одне й те саме медіа вбудовується один раз, скільки б разів його
  не використали;
- секції (`w:sectPr` у `w:pPr` абзацу + body-рівня) з власними розміром сторінки
  й полями кожна, розриви секцій (нова сторінка; `continuous` продовжує поточну),
  колонки (`w:cols`: `w:num`/`w:space`/`equalWidth` або явні `w:col`) — текст
  переливається в наступну колонку, а з останньої на нову сторінку — явні
  розриви колонки (`<w:br w:type="column"/>`) і роздільна лінія між колонками
  (`w:sep` — вертикальна риска по центру проміжку, на всю висоту колонок);
- колонтитули (`w:headerReference`/`w:footerReference`, частини `word/headerN.xml`
  і `word/footerN.xml`) із повним вмістом (абзаци, таблиці, зображення, списки),
  малюються на кожній сторінці своєї секції на відступах `w:pgMar` `w:header`/
  `w:footer` (футер прив'язаний нижнім краєм, тож багаторядковий росте вгору);
  `w:titlePg` дає окремі колонтитули першої сторінки;
- поля номера сторінки й гіперпосилання: `PAGE` показує номер поточної сторінки,
  `NUMPAGES` — загальну кількість сторінок (обидва в простій формі `w:fldSimple`
  й у складній `w:fldChar`/`w:instrText`), причому значення обчислюється під час
  рендеру, а не береться кешованим із файлу, тож у колонтитулі виходить живий
  «Сторінка 3 із 8»; інші поля показують свій кешований у документі результат.
  Текст гіперпосилань (`w:hyperlink`) відтворюється на своєму місці в абзаці
  (з синім підкресленням, якщо документ посилається на стиль «Hyperlink»);
- автоматичне перенесення по словах і посторінкова розбивка;
- збережені пробіли (`xml:space="preserve"`): послідовності пробілів зберігають
  повну ширину, а провідні пробіли створюють відступ першого рядка для абзаців
  з вирівнюванням ліворуч/за шириною, тож відступи, задані пробілами (типово, коли
  документ не має `w:ind`), збігаються з Word, а не згортаються в один пробіл;
- геометрія сторінки з кожної секції (`w:sectPr`): розмір сторінки (`w:pgSz`)
  і поля (`w:pgMar`) визначають ширину контенту, точки переносу й розриви сторінок,
  тож переноси рядків відповідають джерелу, а не фіксованій сторінці, а сторінки
  різних секцій можуть бути різного розміру (напр. альбомна вставка). За
  відсутності — запасний варіант A4 з полями 1 дюйм;
- текст у Unicode, зокрема кирилиця — відтворюється вбудованими шрифтами (SIL OFL
  1.1) замість латинських core-шрифтів PDF, тож нелатинські системи письма дають
  коректні гліфи, а не «кракозябри». Кожен шрифт метрично-сумісний із тим, що його
  оголошує Word, і водночас покриває кирилицю: Liberation Sans/Serif/Mono
  (Arial/Times New Roman/Courier New) і Carlito (Calibri — типовий шрифт тіла
  сучасного Word), тож переноси рядків і кількість сторінок лишаються близькими до
  Word. Метричний клон Cambria (Caladea) навмисно не вбудовано — він без кирилиці,
  тож малював би її порожніми прямокутниками; Cambria мапиться на Liberation Serif;
- таблиці: рядки/комірки, ширини колонок (`w:tblGrid`), горизонтальне (`gridSpan`)
  і вертикальне (`vMerge`) об'єднання комірок, рамки по сторонах, по внутрішніх
  лініях сітки (`w:insideH`/`w:insideV` — беруться лише на межах між комірками, а
  по краю таблиці стоїть її зовнішня рамка) і по діагоналях (`w:tl2br`/`w:tr2bl`),
  заливка комірок, вкладені таблиці та посторінкова розбивка
  (рядок ніколи не розривається між сторінками). Рамки/заливка резолвляться спершу
  з inline `w:tblPr`/`w:tcPr`, а далі — з іменованого стилю таблиці (`w:tblStyle`,
  через `styles.xml`, за `w:basedOn`), включно з умовними регіонами `w:tblStylePr`
  (смугасті рядки/колонки, перший/останній рядок/колонка, кутові комірки
  `nwCell`/`neCell`/`swCell`/`seCell` — кут перекриває і рядок, і колонку) під
  керуванням `w:tblLook`, тож таблиця, оформлена у Word звичайним способом (напр.
  вбудований «Table Grid» чи смугастий стиль), відображається з видимою сіткою й
  заливкою регіонів, а не лише коректно розташованим текстом. Регіони дають ще й
  оформлення абзаців і тексту (`w:pPr`/`w:rPr` всередині `w:tblStylePr` — напр.
  жирний центрований рядок заголовка): це окремий рівень успадкування, під
  іменованими стилями абзацу/символів і над `w:docDefaults`. Власний лівий відступ
  таблиці (`w:tblInd`) резолвиться так само (спершу inline, потім її іменований
  стиль) і зсуває її від поля сторінки (а для вкладеної таблиці — від краю контенту
  батьківської комірки);
- ширини колонок, які документ не задає (немає `w:tblGrid`, `w:tcW` типу `auto`),
  підбираються за вмістом: колонка просить стільки, скільки займає її найдовший
  нерозірваний рядок, але гарантовано дістає щонайменше своє найдовше слово, а те,
  що лишили задані колонки, ділиться пропорційно до решти запиту. Комірки, що
  охоплюють кілька колонок (`gridSpan`), розширюють їх, коли ті разом вужчі за
  потрібне. Відсоткові ширини (`w:tcW` типу `pct`, зокрема у 50-х частках відсотка)
  беруться від доступної ширини, а `w:tblLayout w:type="fixed"` вимикає підбір
  зовсім: задані ширини стоять, а незадані ділять залишок порівну. Якщо таблиця
  задає власну ширину (`w:tblW` у dxa або відсотках), колонки масштабуються рівно
  під неї;
- відступи в комірках (`w:tcMar` на комірці, `w:tblCellMar` на таблиці й у її
  стилі, з типовими значеннями Word: 108 dxa ліворуч/праворуч, 0 згори/знизу;
  сторона у відсотках береться від ширини таблиці), вертикальне вирівнювання
  вмісту (`w:vAlign` — згори/по центру/знизу) і задана висота рядка (`w:trHeight`):
  `atLeast` робить її мінімумом, `exact` — фіксує, і зайвий вміст обрізається по
  комірці, як у Word, а не наповзає на наступний рядок;
- повторювані рядки-заголовки (`w:tblHeader`) — перші такі рядки перемальовуються
  вгорі кожної сторінки чи колонки, куди переходить таблиця;
- вертикальний напрям тексту в комірці (`w:textDirection` `btLr`/`tbRl`): текст
  обертається на 90°, переноситься по довжині рядка (за заданою висотою рядка або
  за тим, що лишилося на сторінці), а висота рядка росте під нього;
- об'єднана по вертикалі комірка (`vMerge`), вміст якої вищий за охоплені нею
  рядки, розсуває їх: нестача ділиться між тими, що можуть рости (рядок фіксованої
  висоти лишається як є), пропорційно до їхньої висоти;
- плаваючі таблиці (`w:tblpPr`): позиція рахується від тексту, поля чи краю
  сторінки (`w:horzAnchor`/`w:vertAnchor`) — або зсувом (`w:tblpX`/`w:tblpY`), або
  вирівнюванням (`w:tblpXSpec`/`w:tblpYSpec`); така таблиця не рухає курсор тексту.

Поки не обробляється (непідтримуваний вміст **пропускається**, конвертація все одно
завершується успішно):

- виноски; поля Word поза `PAGE`/`NUMPAGES` (показують лише кешований у файлі
  результат); клікабельність гіперпосилань — текст рендериться, але без активного
  переходу (анотація-посилання в PDF ще не додається);
- зі зображень — формати, яких не вміє PDF-бекенд (EMF/WMF/SVG/TIFF: такий рисунок
  просто пропускається), обтікання текстом і абсолютне позиціювання плаваючих
  рисунків (`wp:anchor` малюється як вбудований), обрізка й ефекти;
- з колонтитулів — окремі парні/непарні (`w:evenAndOddHeaders`); високий
  колонтитул не зсуває тіло сторінки, а накладається на поле;
- з колонок — балансування висоти на `continuous`-секції (колонки заповнюються
  згори вниз, а не вирівнюються по висоті);
- шрифти поза вбудованим набором (шрифти Word — зокрема ті, що прийшли з темою, —
  мапляться на метрично-сумісні заміни: Calibri → Carlito, решта serif (зокрема
  Cambria) → Liberation Serif, monospace → Liberation Mono, інші → Liberation
  Sans, за широким, але скінченним списком ключових слів, тож переноси рядків і
  кількість сторінок можуть трохи відрізнятися від Word; без вбудовування самих
  шрифтів Word чи пошуку в ОС);
- з таблиць — обтікання текстом навколо плаваючої таблиці (вона малюється на
  своєму місці, але текст не розступається й може пройти під нею) і відстані до
  тексту (`w:leftFromText` тощо); плаваюча таблиця не переноситься на наступну
  сторінку, якщо не влазить.
